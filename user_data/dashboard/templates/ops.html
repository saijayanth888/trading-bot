How these parameters work — read before changing valuesdecision guide

The 5 market regimes

A 4-state HMM classifies each candle into one of these regimes per pair. The strategy adapts entries, exits, sizing, and trailing-stop behaviour to the active regime.

trending_up
Sustained uptrend. Strategy: loosen entries, hold longer, activate trailing stop on winners.
trending_down
Sustained downtrend. Strategy: longs are hard-blocked — bot waits for regime change. The entry_delta here is belt-and-suspenders.
mean_reverting
Range-bound, oscillating market. Strategy: quick scalps with tight take-profit (mean_rev_take_profit).
high_volatility
Whippy, hard-to-predict. Strategy: shrink position size (high_vol_stake_factor) and require higher conviction (high_vol_min_confidence).
unknown
HMM uncertain. Strategy: conservative defaults — neither blocked nor preferred.

Entry & exit deltas

These add a per-regime offset to the base thresholds. Base entry = 0.62 (TFT up-probability needed to fire a long); base exit = 0.55 (down-probability needed to close).

entry_delta = +0.15
Require up_prob ≥ 0.62 + 0.15 = 0.77. Harder to enter in this regime.
entry_delta = −0.05
Require up_prob ≥ 0.62 − 0.05 = 0.57. Easier to enter.
entry_delta = blank
Hard-block — no longs allowed in this regime. Same as setting threshold to ∞.
exit_delta = −0.20
Require down_prob ≥ 0.55 − 0.20 = 0.35. Faster exits — close on weaker signals.
exit_delta = +0.05
Require down_prob ≥ 0.60. Hold longer, only exit on strong reversal.

Scalar parameters

high_vol_stake_factor
In high_volatility, multiply position size by this. Default 0.7 (30% smaller). Set 0.5 for half-size, 0 to skip entries entirely. Lower if drawdowns spike in volatile markets.
high_vol_min_confidence
In high_volatility, require up_prob ≥ this on top of the regular threshold. Default 0.65. Higher = fewer but higher-conviction trades.
mean_rev_take_profit
In mean_reverting, exit immediately when profit reaches this fraction. Default 0.012 = +1.2%. Lower = quicker scalps; higher = let winners run further.
trending_up_trail_trigger
In trending_up, when profit exceeds this, activate trailing stop. Default 0.025 = 2.5%. Lower = trail sooner (lock in smaller wins); higher = wait for bigger wins before trailing.
trending_up_trail_distance
Once trailing is active, trail this far below the high-water mark (must be negative). Default −0.02 = 2% below peak. More negative (e.g. −0.03) = wider trail, more room for noise; closer to 0 = tighter trail, gives back less but stops out sooner.
tft_min_confidence
TFT model's quantile-spread confidence floor. Default 0.35. Below this, no entries fire in any regime. Raise to 0.45+ to filter out low-conviction signals at the cost of fewer trades.
meta_min_confidence
When the DRL meta-agent (PPO + A2C + DQN ensemble) is active, require this confidence on the meta_signal. Same logic for entries (signal=+1) and exits (signal=−1). Default 0.35.

Recommended tuning order

  1. Start with defaults — they're calibrated to work end-to-end.
  2. Too many losing entries: raise tft_min_confidence (+0.05 increments) or meta_min_confidence.
  3. No trades firing: lower tft_min_confidence, then check that no allowed regime has entry_delta = blank.
  4. Drawdowns in volatile markets: drop high_vol_stake_factor to 0.4, raise high_vol_min_confidence to 0.8.
  5. Profits get given back in trends: tighten trending_up_trail_distance closer to 0 (e.g. −0.015).
  6. Whipsawing in chop: raise mean_rev_take_profit to 0.018+ to ignore tiny moves.
Apply changes writes config.json atomically (with timestamped backup) and triggers a freqtrade reload — the bot keeps running, but new candles will use the updated values. Open trades are not affected mid-flight; only future entries/exits use the new parameters.
